# Choice Tournament PRD

## 1) 제품 개요
**choice-tournament**는 사용자가 주제와 8개, 16개 또는 32개 후보를 입력하고, 토너먼트 방식으로 하나씩 선택해 최종 우승 1개를 도출하는 정적 웹앱입니다.

## 2) 목표
- 입력부터 결과까지 **1분 내 완료** 가능한 빠른 의사결정 경험 제공
- 모바일 우선의 간단한 UX 제공
- 대진 설정을 **링크로 공유**할 수 있도록 정적 호스팅 친화적으로 설계

## 3) 비목표 (MVP 범위 제외)
- 로그인/계정 시스템 및 서버 DB
- 실시간 멀티플레이(동시 투표)
- 서버 업로드/동기화가 필요한 미디어 저장

## 4) 사용자 플로우
1. 홈에서 새 토너먼트 시작
2. 주제 + 후보 8개/16개/32개 입력
3. 라운드 옵션(8강/16강/32강) 선택 후 매치 진행(2개 중 1개 선택)
4. 32강 선택 시: 32강 → 16강 → 8강 → 준결승 → 결승
5. 16강 선택 시: 16강 → 8강 → 준결승 → 결승
6. 결과(우승/준우승/4강) 확인 및 공유/재시작

## 5) 핵심 사용자 스토리
- 사용자는 주제와 후보 8개/16개/32개를 붙여넣기로 빠르게 입력할 수 있다.
- 사용자는 후보별 로컬 이미지를 첨부할 수 있으며, 이미지가 자동으로 리사이즈/압축되어 저장된다.
- 사용자는 한 화면에서 두 후보 중 하나를 탭해 즉시 다음 매치로 넘어갈 수 있으며, 라운드 시작 시 전체화면 타이틀 애니메이션으로 단계 전환을 인지할 수 있다.
- 사용자는 실수한 선택을 최소 1회 이상 되돌릴 수 있다.
- 사용자는 새로고침/앱 재진입 후에도 진행 중 토너먼트를 이어갈 수 있다.
- 사용자는 링크로 주제/후보/시드 설정을 공유할 수 있다.
- 사용자는 최종 결과와 대진표를 확인할 수 있다.

## 6) 성공 지표 (MVP)
- 생성 화면에서 결승 결과까지 도달률
- 평균 완료 시간(목표: 60~90초)
- 공유 링크 생성률
- 모바일 화면별 이탈률

## 7) 접근성/사용성 요구사항
- 모바일 우선 UI
- 키보드 단축키 지원(선택 1/2, 되돌리기)
- 터치 타겟 최소 44px 권장

## 8) 품질 체크리스트 (MVP)
- 후보 8개/16개/32개 검증(공백/빈 줄 처리)
- 시드 기반 셔플 재현 가능성
- 라운드 진행 정확성(32→16→8→4→2 또는 16→8→4→2)
- Undo 동작(최소 1회)
- localStorage 기반 재개
- 로컬 이미지 처리(최대 256x256 리사이즈, base64 문자열 저장) 검증
- 공유 링크 생성/복원
- 모바일 카드 클릭/스크롤 이슈 점검
- GitHub Pages 배포 후 라우팅 점검

## 9) 확장 백로그
- 후보 이미지 URL 지원
- 결과 이미지 저장
- 프리셋 템플릿(점심/여행지/영화 등)
- 3~4위전 옵션
- 반복 실행 통계
- 멀티 유저 투표 모드(서버 필요)
